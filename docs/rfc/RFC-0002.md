## 3. Phase Rules

### [RFC-0002:C-PHASE-ORDER] Phase Ordering (Normative)

Phases MUST proceed in order: `spec → impl → test → stable`.

A feature in `impl` phase MUST have a normative RFC that has passed `spec` phase.
A feature in `test` phase MUST have passed `impl` phase gate.
A feature in `stable` phase MUST have passed `test` phase gate.
